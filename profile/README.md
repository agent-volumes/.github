# Agent Volumes Organization

The Agent Volumes Organization develops open specifications for packaging, distributing, and verifying AI agent
components across runtimes, registries, package managers, and developer tools.

Agent Volumes is a Founder-led, TSC-first open specification project. The specification is vendor-neutral and
implementation-agnostic: no runtime, registry, hosted service, vendor, or downstream implementation defines the standard.

## Governance

Agent Volumes is led by [Yunseo Kim](https://github.com/yunseo-kim) as Founder and Project Lead, with a provisional
Technical Steering Committee (TSC) as the primary deliberative and technical review body.

- The [Charter](https://github.com/agent-volumes/.github/blob/main/CHARTER.md) defines mission, scope, conformance
  boundaries, and Founder/TSC authority.
- The [Governance](https://github.com/agent-volumes/.github/blob/main/GOVERNANCE.md) document explains roles, decision
  flow, Founder approval, TSC review, maintainers, Working Groups, and amendments.
- Governance proposals and TSC participation questions belong in
  [`agent-volumes/.github`](https://github.com/agent-volumes/.github/issues/new/choose).

## Participate

- **Contributors:** review the [contribution guidelines](https://github.com/agent-volumes/.github/blob/main/CONTRIBUTING.md)
  and open issues or pull requests in the repository that owns the artifact.
- **Implementers:** use the [Implementers Guide](https://github.com/agent-volumes/agent-volumes-spec/blob/main/IMPLEMENTERS.md)
  when building compatible runtimes, clients, registries, validation tools, or packaging workflows.
- **Adopters and evaluators:** if your organization uses or is evaluating Agent Volumes, consider
  [listing it in ADOPTERS.md](https://github.com/agent-volumes/agent-volumes-spec/blob/main/ADOPTERS.md).
- **Organizations interested in deeper participation:** open a governance discussion or issue about TSC participation,
  Working Groups, implementation feedback, or future member-company participation.
